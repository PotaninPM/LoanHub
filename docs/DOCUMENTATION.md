# Technical Documentation - LoanHub

> [!IMPORTANT]
> ⚠️ **Внимание**: Просьба тестировать на физическом устройстве или на эмуляторе, но с включенным ВПН. По причинам санкций с эмулятора без ВПН запрос к Supabase может не доходить (на физическом устройстве ВПН не нужен).

В данном документе приведена детальная техническая информация об устройстве проекта LoanHub.

## 🏗 Архитектура системы

Приложение построено с использованием **Clean Architecture**. Проект разделен на слои для обеспечения тестируемости и гибкости.

### Модульная структура
| Модуль | Описание                                                                          |
| :--- |:----------------------------------------------------------------------------------|
| `:app` | Главный модуль.                                                                   |
| `:sources:features:*` | Функциональные модули. Содержат UI, ViewModels и доменную логику конкретной фичи. |
| `:sources:common:uikit` | Дизайн-система проекта. Компоненты, стили, анимации.                              |
| `:sources:common:network` | Интеграция с Supabase, Retrofit клиенты, общие DTO.                               |
| `:sources:common:database` | Описания сущностей Room и локальная БД.                                           |
| `:sources:core:*` | Навигация, авторизация, базовые типы и утилиты.                                   |

## 🎨 UI Kit и Визуальная часть

Приложение полностью построено на **Jetpack Compose** — современном декларативном фреймворке от Google. Это позволило нам реализовать сложную дизайн-систему с высокой степенью гибкости.

Все визуальные элементы вынесены в отдельную библиотеку компонентов:
- **LoanHubUiTextField**: Интегрированная валидация и поддержка подсказок (Dadata).
- **Haze (Glassmorphism)**: Библиотека Haze используется для создания эффекта размытия заднего плана. Мы выбрали ее вместо стандартного модификатора `blur`, так как стандартное решение не обеспечивает качественного эффекта "стекла" и имеет ограничения на различных версиях API.
- **Shimmer Effects**: Унифицированная система скелетонов для всех типов контента.
- **Gradients & Icons**: Динамическая раскраска карточек кредитов на основе данных из БД.

## 🛠 Работа с данными (Data Layer)

### Supabase & Retrofit Integration
Мы используем Supabase для хранения всех данных о заявках, кредитных продуктах и профилях пользователей. Обмен данными происходит через **Retrofit**, что гарантирует высокую скорость и надежность сетевых запросов.
- **Auth**: Используется почта/пароль. Токены сохраняются в зашифрованных Prefs.
- **BDUI**: Список доступных кредитов загружается из таблицы `loan_products`, что позволяет менять контент без пересборки APK.

### Локальное хранение и Черновики
Используется **Room Database** для кэширования и работы в оффлайн-режиме:
1. Текущий прогресс по заявке сохраняется в локальную БД.
2. При открытии экрана оформления, данные подтягиваются из профиля или последнего черновика.
3. Незавершенные заявки отображаются на главном экране в виде горизонтальной ленты.

## 🛡 Тестирование

Проект придерживается пирамиды тестирования для обеспечения стабильности при рефакторинге:
1. **Unit Tests**: Покрытие бизнес-логики (UseCase) и мапперов.
   - **Стек**: JUnit 4, MockK (мокирование), Kotlinx Coroutines Test.
   - Запуск: `./gradlew test`
2. **Component Tests**: Проверка взаимодействия UI компонентов и вьюмоделей (например, фильтрация заявок).
   - **Стек**: Robolectric, Compose UI Test Library.
3. **Static Analysis**: Кастомные правила Detekt для соблюдения coding style.
   - Запуск: `./gradlew detekt`
