# LoanHub 💰

> [!IMPORTANT]
> ⚠️ **Внимание**: Просьба тестировать на физическом устройстве или на эмуляторе, но с включенным ВПН. По причинам санкций с эмулятора без ВПН запрос к Supabase может не доходить (на физическом устройстве ВПН не нужен).

**LoanHub** — это современное Android-приложение для агрегации и оформления кредитных заявок (потребительские кредиты, ипотека, автокредиты, лизинг). Проект построен на масштабируемой многомодульной архитектуре и интегрирован с облачной инфраструктурой Supabase.

---

## 🚀 Основные возможности (Extra Features)

Мы пошли дальше базовых требований и реализовали ряд продвинутых функций для лучшего UX:

- **Связь с Supabase**: Все приложение напрямую связано с Backend(Supabase), все заявки, типы кредитов, профиль юзера - все это уходит/приходит на/с бэкенд(Supabase).
- **Экран авторизации**: В проекте есть авторизация для того чтобы все сохранялось на бэкенде и пользователь смог не беспокоится о потери данных.
- **Система Access/Refresh токенов**: Авторизация работает на системе Access, Refresh токенов, обеспечивая максимальную безопасность.
- **Интеллектуальная Админка**: Встроенный режим администратора для управления потоком заявок (Approve/Reject). Чтобы зайти под админом выберите тестовый аккаунт админа.
- **Лизинг**: Добавлен новый полноценный тип финансового продукта с уникальным набором полей.
- **Черновики (Drafts)**: Система автоматического сохранения незаполненных заявок с отображением на главном экране.
- **Детали кредита**: Есть полноценный экран деталей кредита, чтобы пользователь мог посмотреть полную информацию по кредиту.
- **Профиль**: Полноценный экран профиля со всей информацией о пользователе.
- **Популярные предложения**: На экране списка кредитов есть Pager, который можно листать, данные оттуда приходят от бэка из отдельной таблички, можно регулировать очередь карточек и всю информацию на карточках и их цвет.
- **Система автозаполнения заявки**: Система, которая позволяет подтягивать данные из профиля сразу в заявку (показывается диалоговое окно на экране заявки).
- **Редактирование профиля**: Экран редактирования профиля с возможностью редактирования. Данные из профиля автоматически подтягиваются в новые заявки.
- **BDUI Elements**: Динамическое управление списком кредитов (названия, градиенты, иконки, порядок показа) через Remote Config (Supabase).
- **Rich UI**: Кастомная дизайн-система, плавные анимации перехода, продвинутые шиммеры и скелетоны для всех состояний загрузки.
- **Unit-тесты**: Каждая фича покрыта юнит-тестами.
- **Компонентные тесты**: Создана среда и написаны компонентные тесты на логику фильтров с использованием Robolectric.
- **Gitlab CI**: Настроен Gitlab CI на котором прогоняются Detekt и юнит тесты.

## 🛠 Технологический стек

- **Язык**: Kotlin
- **UI**: Jetpack Compose
- **Архитектура**: Clean Architecture + Multimodule (Manual DI)
- **Бэкенд**: Supabase (PostgreSQL, Auth)
- **Сетевой слой**: Retrofit + OkHttp + Kotlin Serialization
- **Локальное хранение**: Room Database
- **UI Effects**: Haze (Advanced Glassmorphism)
- **Image Loading**: Coil
- **Качество кода**: Detekt, Unit Tests, Component Tests
- **CI/CD**: GitLab CI
- **Git**: Использовались Conventional Commits и Merge requests

---

## 📦 Структура проекта

Проект разделен на логические слои:
- `:app` — точка входа.
- `:sources:features` — продуктовые независимые функциональные модули (Auth, LoanApplication, MyRequests, Profile и др.).
- `:sources:common` — общие компоненты (UI Kit, Network, Database, DI).
- `:sources:core` — навигация, авторизация, базовые интерфейсы и утилиты.

## 🛠 Сборка и запуск

1. Склонируйте репозиторий.
2. Откройте проект в Android Studio Ladybug+.
3. Настройте `local.properties` (при необходимости).
4. Запустите сборку: `./gradlew assembleDebug` (желательно собрать релизную сборку для полноценного ощущения кайфа)

> [!TIP]
> **Релизная сборка**: Вы можете найти готовый APK файл для установки в папке проекта по пути: `app/release/app-release.apk`.

---

## 🧪 Качество и тесты

- Выполните проверку кода: `./gradlew detekt`.
- Запустите тесты: `./gradlew test`.
- Каждый Merge Request проходит проверку в CI.
